La anotación @OneToMany en la entidad Competitor establece una relación entre esta entidad y otra, en este caso Producto. Esto tiene un impacto directo en la base de datos, ya que se crea una tabla adicional que mantiene una relación entre la tabla de Competitor y la tabla de Producto. La anotación mappedBy="competitor" indica que la propiedad competitor en la entidad Producto es la que posee la clave foránea que vincula ambos modelos. En términos prácticos, la relación genera una estructura de base de datos en la que cada competidor puede estar asociado a varios productos, pero cada producto pertenece a un único competidor
